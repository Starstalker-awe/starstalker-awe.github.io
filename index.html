Tag Syntax Description
base 1: <base target="[_self|_blank|_parent|_top]">
2: <base target="[option]" href="{url}">
3: <base href="{url}"> 		The base tag is used to create a base url for the document, unless an absolute url is specified. Any url specified that begins with only a / will be appended to the base url. The target attribute also specifies a default target. The url can be overriden by using an absolute url.
form <form action="{url}" [method="[post|put|...]|get"]>
 {input fields}
</form> 		Forms are used, as well, forms... They aren't completetly necessary for submitting user inputted data to a server, but they make a very useful container that also allow you to specify the submit url and method if the data isn't submitted through JavaScript.
input <input (type="[file|email|...]|text") name="{name}"> The input element is used for collecting data, typically contained within a form. There are many different types, such as color, password, and checkboxes and radio buttons for selecting. A full list can be found on MDN.
menu <menu>
 {li items}
</menu> 		The menu element is a semantic HTML tag for menus, but is still constructed using or <li> tags.
pre <pre>
 {content}
</pre> 		The pre tag is typically used to share a code snippet. All content is escaped, and whitespace is preserved exactly.
